capabilities

what re/up does

re/up runs a local server on your machine that your roblox studio plugin talks to. it checks permissions, fetches asset data, and reuploads everything automatically.

sound reupload

batch reuploads audio assets using a fixed window rate limiter. supports multiple accounts so when one hits its roblox upload quota or gets rejected, re/up automatically rotates to the next account in the pool.

mesh reupload

reuploads meshes with correct joints, attributes, tags, and children preserved. up to 3000 uploads per minute. if a name is flagged as inappropriate it gets marked as censored and continues.

animation reupload

reuploads animations at up to 420 per minute with 24 concurrent uploads. handles rate limit backoff automatically so if roblox returns a retry-after header, re/up waits exactly that long before retrying.

smart filtering

re/up skips assets that are already yours or already accessible from your place. only assets that actually need reuploading get processed, nothing is done twice.

multi-account pool

add multiple roblox accounts in the desktop app. re/up rotates through them automatically when one gets rate limited or hits its quota, keeping uploads running without interruption.

cancel and pause

cancel a job mid-run and choose whether to keep the ids already uploaded or discard them. pause and resume without losing progress. the server stays alive after a job finishes so you can rerun immediately.

video tutorial

watch it in action

follow along with the full walkthrough. covers the whole setup from download to first successful reupload.

step 01

download and install

grab the installer from the github releases page and run it. both the desktop app and the studio plugin install automatically, nothing to configure separately.

step 02

paste your cookie and api key

open re/up and paste your .ROBLOSECURITY cookie and your open cloud api key. the api key needs assets read and write plus asset permissions write at create.roblox.com/dashboard/credentials.

step 03

enable studio settings

in roblox studio go to file then studio settings then security and turn on allow http requests. then go to file then beta features and turn on lua asset creation api. restart studio after both.

step 04

open your place and connect

paste your place id in the app and hit launch studio. the app shows connected in green when the plugin is talking to the backend. if it doesnt connect after 30 seconds check that http requests is on and restart studio.

step 05

start reuploading

pick your asset type in the plugin toolbar and hit start reuploading. the plugin scans the place, re/up handles the rest. when it finishes save the place with ctrl+s or your ids wont stick.

in minutes

everything comes bundled in the installer. no separate plugin download, no config files to hunt down.

got errors? open the error assistant below for step-by-step fixes, or hit up the discord. im available at simplykur on discord.
01

download and install

download the installer from the github releases page and run it. the desktop app and the roblox studio plugin are both installed automatically.

02

open the desktop app

launch re/up. paste your .ROBLOSECURITY cookie into the app. if you want multi-account support, add additional cookies here too.

03

open roblox studio

the plugin is already installed. open your game, find the re/up plugin in the toolbar, and select the asset type you want to reupload.

04

hit reupload

re/up checks permissions, fetches your assets, filters out anything already accessible, and reuploads everything to your account or group automatically.

05

rerun without restarting

when a job finishes the server stays alive. just hit reupload again from the plugin without touching the desktop app.
