capabilities

what re/up does

re/up runs a local server on your machine that your roblox studio plugin talks to. it checks permissions, fetches asset data, and reuploads everything automatically.

sound reupload

batch reuploads audio assets using a fixed window rate limiter. supports multiple accounts so when one hits quota, re/up automatically rotates to the next in the pool.

mesh reupload

reuploads meshes with correct joints, attributes, tags, and children preserved. up to 3000 uploads per minute. censored names get flagged and continue without blocking the job.

animation reupload

reuploads animations at up to 420 per minute with 24 concurrent uploads. handles rate limit backoff so if roblox returns retry-after, re/up waits exactly that long before retrying.

smart filtering

re/up skips assets already yours or accessible from your place. checks permissions before starting so you never waste time uploading to a place you cannot edit.

multi-account pool

add multiple roblox accounts. re/up rotates through them automatically when one gets rate limited or hits its quota, keeping uploads running without interruption.

cancel and pause

cancel mid-run and choose whether to keep or discard uploaded ids. pause and resume without losing progress. the server stays alive after a job finishes so you can rerun immediately.

video tutorial

watch it in action

follow along with the full walkthrough. covers the whole setup from download to first successful reupload.

step 01

download and install

grab the installer from github releases and run it. both the desktop app and the studio plugin install automatically, nothing to configure separately.

step 02

paste your cookie and api key

open re/up and paste your .ROBLOSECURITY cookie and open cloud api key. the key needs assets read and write plus asset permissions write at create.roblox.com/dashboard/credentials.

step 03

enable studio settings

in roblox studio go to file, studio settings, security and turn on allow http requests. then file, beta features and turn on lua asset creation api. restart studio after both.

step 04

open your place and connect

paste your place id in the app and hit launch studio. the app shows connected in green when the plugin is talking to the backend. if it doesnt connect after 30 seconds restart studio.

step 05

start reuploading

pick your asset type in the plugin toolbar and hit start reuploading. when it finishes save the place with ctrl+s or your ids wont stick.

error reference

common errors and fixes

re/up detects these automatically and shows the fix in the in-app assistant. if you see one, here is what it means and what to do.

account does not have permission to edit place
the account you are uploading with is not an editor of the target place. either the place is owned by a different user or group, or the account was not given edit access. re/up stops early instead of uploading 0 assets.
add account as editor
cookie auth failed / 401
your .roblosecurity cookie has expired or is invalid. this usually happens after a password change or when roblox rotates security tokens. re/up marks the account as invalid and skips it.
re-paste cookie in app
403 forbidden on upload
roblox rejected the upload request. most common cause is the open cloud api key not having the required scopes: assets.read, assets.write, and asset-permissions.write on the correct creator.
check api key scopes
animation failed to load
the source animation asset could not be fetched from roblox before reuploading. usually a deleted, private, or restricted asset. re/up logs it, skips it, and continues the rest of the job.
check source asset exists
rate limited / retry-after
roblox returned a rate limit header. re/up reads the retry-after value and waits exactly that many seconds before resuming. no action needed, it resolves automatically.
wait, auto-recovers
name rejected / content policy
the asset name was flagged by roblox content filters. re/up marks the asset as censored, uses a fallback name, and continues uploading. the asset still gets reuploaded successfully.
auto-handled
plugin not connecting
the studio plugin cannot reach the re/up backend on port 38073. either http requests is off in studio settings, the backend crashed, or a firewall is blocking localhost connections.
enable http requests in studio
upload quota exceeded
the current account hit roblox daily upload limits. re/up rotates to the next account in the pool automatically. if all accounts are exhausted, the job pauses and shows the limit message.
add more accounts to pool
